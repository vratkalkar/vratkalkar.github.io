Dynamic vs Static Typing

November 30th, 2014

When you first hear about dynmaic and static typing, your first thought might be, "Oh, that has to be how you type on the keyboard. Right?" Well, that's quite literal and is not the case.

Type Systems

In programming speak, "typing" refers to data types. Also, there is what is called a type system. A type system is a set of rules that assign a type to variables, functions or expressions. The goal of a type system is to eradicate bugs in your program by defining interfaces between the parts of the program. Then the parts are checked to ensure that the connection is consistent.

The checking of the parts can happen at compile time(static), or run-time(dynamic) meaning when the program is running/executed

Static Typed

As mentioned earlier, when the type check is done when compiling, the language is said to be static. Type information is bound to a variable and is checked at compile time. Here are a few static typed languages for your reference:

  • Java
  • C
  • Scala

Dynamic Typed

A programming language is said to use dynamic typing, if the type check is done at run-time. In this case, the type of a variable is bound to its value and type checking occurs at run time

Here a few of todays widely used languages that use dynamic typing:

  • Ruby
  • JavaScript
  • Python
  • PHP
