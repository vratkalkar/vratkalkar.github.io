There are times when you want to create a set of methods to run without individually defining a method and calling it on an object.

This is where classes become beneficial, if not necessary.

A class in ruby sets a constant, or class name followed by defined methods. To provide a visual here is an example of a class named Number:

  • This class is initialized with a passed argument "(num)" and sets it as an instance variable, "@num".
  • The instance variable allows that set "num" value to be accessed across all methods in the class.
  • The subsequent methods access the "@num" value to perform the arithmetic defined in the method

Now you're probably thinking, "That's great, but how do you set the number!?". That is where creating an instance of the class comes in, which you can see on line 19 f the gist above.

In the example I defined "example" which is equal to a new instance of the Number class set to 10, "Number.new(10)". The initialize method sets the instance variable to 10 and "example" can be called on each method as it is an object of the class.

This is a quick overview, and as always I recommend researching when possible to get a better understanding of the concept of classes in Ruby.
